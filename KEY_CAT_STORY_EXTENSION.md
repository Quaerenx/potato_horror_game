# 목에 열쇠를 단 고양이 스토리 확장안

## 한 줄 소개

편의점 앞에서 빠져나오지 못하던 주인공이 목에 낡은 열쇠를 단 고양이를 만나고, 고양이를 따라가며 편의점의 잠긴 문과 어둠의 정체에 한 걸음 더 가까워지는 공포 확장 구간.

## 추가 의도

- 기존의 `편의점은 안전지대가 아니라 미끼`라는 핵심 반전을 유지한다.
- 새 위협을 무작정 늘리기보다, 고양이를 단서와 유도 장치로 사용한다.
- 귀여운 동물 에셋과 공포 분위기의 대비로 여자친구가 기억하기 쉬운 장면을 만든다.
- 플레이타임을 약 2~3분 늘리되, 전투나 복잡한 시스템은 추가하지 않는다.

## 신규 캐릭터

### 열쇠 고양이

목에 낡은 황동 열쇠를 달고 있는 크림색 고양이. 편의점 주변에서 갑자기 나타나 주인공을 바라보다가, 일정 거리 이상 가까워지면 천천히 골목 쪽으로 걸어간다.

겉보기에는 귀엽고 안전해 보이지만, 고양이가 지나간 자리마다 편의점 불빛이 한 번씩 깜빡인다. 열쇠는 편의점 문을 여는 물건처럼 보이지만, 실제로는 주인공을 더 깊은 어둠으로 끌어들이는 미끼인지도 모른다.

사용 에셋:

```text
res://assets/source/animals/key-cat-single.png
```

## 스토리 배치 위치

추천 삽입 위치:

```text
편의점 도착
-> 단서 조사 2개 완료
-> 열쇠 고양이 등장
-> 고양이를 따라 짧은 골목 조사
-> 목걸이 열쇠 확인
-> 편의점 문 조사 가능
-> 최종 추격
```

기존 흐름을 크게 바꾸지 않으려면, 고양이는 `STORE_REACHED` 단계 안에서 발생하는 추가 이벤트로 둔다.

## 진행 구조

1. 주인공이 편의점 주변 단서를 2개 이상 조사한다.
2. 자동문 센서가 붉게 깜빡이고, 편의점 옆 골목에서 작은 방울 소리가 난다.
3. 목에 열쇠를 단 고양이가 등장한다.
4. 고양이는 주인공을 바라본 뒤 골목 안쪽으로 천천히 이동한다.
5. 주인공이 따라가면 벽에 긁힌 자국, 오래된 사료 그릇, 떨어진 영수증 조각을 발견한다.
6. 고양이가 잠시 멈춰 앉고, 목의 열쇠가 편의점 불빛을 받아 반짝인다.
7. 주인공이 열쇠를 잡으려는 순간 고양이가 사라지고, 열쇠만 바닥에 떨어진다.
8. 열쇠를 주운 뒤 편의점 문을 다시 조사할 수 있다.
9. 문은 열리지 않는다. 대신 안쪽이 아니라 뒤쪽 어둠에서 노크 소리가 들린다.
10. 최종 추격이 시작된다.

## 주요 장면

### 1. 방울 소리

편의점 앞이 지나치게 조용해진다. 냉장고 모터 소리, 가로등 소리, 주인공 발소리가 잠깐씩 사라지고 작은 방울 소리만 들린다.

연출:

- 화면 가장자리에서 고양이 실루엣이 아주 짧게 지나간다.
- 주인공이 가까이 가면 고양이가 사라지지 않고 기다린다.
- 열쇠가 작게 흔들리며 빛난다.

### 2. 골목 유도

고양이는 위험한 존재처럼 달려들지 않는다. 오히려 너무 침착하게 주인공이 따라오기를 기다린다.

플레이 목표:

```text
목표: 열쇠를 단 고양이를 따라가자
```

주의점:

- 고양이를 너무 멀리 놓치면 방울 소리가 끊기고, 검은 형체의 숨소리가 들린다.
- 고양이를 따라가는 동안 직접 추격은 하지 않는다.
- 공포는 이동 속도보다 조용한 불안감으로 만든다.

### 3. 열쇠가 떨어지는 순간

고양이가 편의점 뒤쪽 벽 앞에 앉는다. 벽에는 손톱으로 긁은 듯한 글자가 있다.

벽 문구:

```text
문은 앞에 없어.
```

주인공이 목걸이에 손을 뻗으면 고양이는 소리 없이 사라지고, 열쇠만 바닥에 떨어진다. 열쇠를 줍는 순간 편의점 자동문 센서가 주인공이 있는 골목 쪽으로 돌아간 듯 붉게 깜빡인다.

## 단서 목록

| 단서 | 위치 | 의미 |
| --- | --- | --- |
| 방울 소리 | 편의점 옆 골목 입구 | 고양이 등장 예고 |
| 긁힌 벽 | 골목 안쪽 벽 | 편의점 정문이 함정이라는 암시 |
| 사료 그릇 | 벽 아래 | 누군가 고양이를 계속 불러들였다는 흔적 |
| 떨어진 열쇠 | 고양이가 사라진 자리 | 문을 여는 도구처럼 보이지만 실제로는 최종 추격 트리거 |

## 대사 초안

### 고양이 첫 등장

```text
시스템: 어디선가 작은 방울 소리가 난다.
주인공: 고양이...?
시스템: 고양이 목에 낡은 열쇠가 흔들리고 있다.
주인공: 저 열쇠, 편의점 문 열쇠인가?
```

### 고양이를 따라갈 때

```text
시스템: 고양이는 도망치지 않는다. 내가 따라오는지 확인하듯 멈춰 선다.
주인공: 이상한데... 그래도 저 열쇠가 필요할 것 같아.
시스템: 편의점 불빛이 뒤에서 한 번 깜빡인다.
```

### 열쇠 획득

```text
시스템: 벽에는 손톱으로 긁은 듯한 글자가 남아 있다.
시스템: 문은 앞에 없어.
주인공: 앞에 없으면... 어디에 있다는 거야?
시스템: 고양이가 사라진 자리에 낡은 열쇠만 떨어져 있다.
```

### 문 조사 직전

```text
주인공: 열쇠가 있으니까 이제 열리겠지.
시스템: 열쇠는 편의점 문에 맞지 않는다.
시스템: 대신 등 뒤에서 똑똑, 하고 문 두드리는 소리가 난다.
주인공: ...뒤에 문이 있었어?
```

## 게임플레이 설계

### 핵심 규칙

- 고양이는 적이 아니다.
- 고양이는 플레이어에게 닿아도 피해를 주지 않는다.
- 고양이는 일정 구간마다 멈춰서 플레이어를 기다린다.
- 고양이 이벤트 완료 후 `cat_key_found` 플래그를 켠다.
- `cat_key_found == true`일 때 편의점 문 조사 대사가 바뀐다.

### 권장 플래그

```gdscript
var cat_event_started := false
var cat_key_found := false
var cat_follow_step := 0
```

### 권장 인터랙션 ID

```text
key_cat
cat_scratched_wall
cat_food_bowl
cat_dropped_key
```

### 권장 목표 문구

| 조건 | 목표 |
| --- | --- |
| 고양이 첫 등장 | 열쇠를 단 고양이를 확인하자 |
| 고양이 이동 중 | 고양이를 따라가자 |
| 골목 도착 | 고양이가 멈춘 곳을 조사하자 |
| 열쇠 획득 후 | 편의점 문을 다시 조사하자 |

## 공포 연출

- 고양이는 귀엽게 보이지만 눈을 너무 오래 마주치면 화면이 살짝 어두워진다.
- 고양이가 지나간 자리에 편의점 불빛이 늦게 따라온다.
- 열쇠 소리는 처음엔 맑은 방울 소리지만, 후반에는 금속 긁는 소리처럼 바뀐다.
- 열쇠를 줍는 순간 기존 검은 형체의 숨소리가 아주 가까이서 들린다.

## 엔딩 암시 추가

엔딩에서 남자친구 차 안으로 들어온 뒤, 주인공의 손에 있던 열쇠가 사라진 것을 알 수 있다.

추가 엔딩 문구:

```text
시스템: 주머니 안을 더듬었지만, 열쇠는 없었다.
시스템: 멀어진 편의점 앞에서 작은 방울 소리가 한 번 울렸다.
```

이 문구를 넣으면 고양이가 구조자인지, 함정의 일부인지 모호하게 남길 수 있다.

## 구현 우선순위

1. 고양이 스프라이트를 맵에 배치한다.
2. 편의점 단서 2개 조사 후 고양이를 보이게 한다.
3. 고양이와 상호작용하면 골목 목표를 띄운다.
4. 골목 단서 1~2개를 조사하게 한다.
5. 열쇠 획득 후 편의점 문 대사를 변경한다.
6. 문 조사 시 기존 최종 추격으로 연결한다.

## 최종 효과

이 확장안은 새 몬스터를 추가하지 않고도 플레이어에게 다음 질문을 남긴다.

```text
고양이는 나를 도와준 걸까?
아니면 처음부터 편의점으로 다시 데려가려던 걸까?
```

그 애매함이 이 게임의 따뜻하지만 찝찝한 엔딩과 잘 맞는다.
